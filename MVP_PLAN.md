# 美妆推荐 MVP 计划 (Makeup Recommendation MVP)

## 项目概述

**核心功能：** 用户上传面部照片 → AI 分析并推荐适合的妆容 →（理想情况）生成带妆容的虚拟试妆照

**评分标准对应：**
- 将课堂所学（RAG、LLM、负责任 AI）恰当应用到项目
- 充分说明如何用 AI 构建项目
- 体现负责任 AI 的考量（偏见、隐私、可解释性）

### Phase 1：基础 MVP（必做）

| 模块 | 功能 | 技术选型 |
|------|------|----------|
| **1. 前端** | 照片上传、结果展示、加载状态 | React / Next.js 或 简单 HTML + Tailwind |

## 三、RAG 设计

### 知识库内容建议

将美妆知识整理成文档，向量化存储：

1. **脸型与妆容**
   - 圆脸、方脸、长脸、菱形脸、鹅蛋脸 → 适合的修容、眉形、腮红位置
2. **肤色与色系**
   - 冷皮 / 暖皮 / 中性皮 → 腮红、唇膏、眼影色系
3. **妆容风格**
   - 日常妆、约会妆、职场妆、派对妆等 → 适用场景与要点
4. **五官强化**
   - 单眼皮、双眼皮、唇形等 → 化妆技巧

### RAG 流程

```
用户照片 → Vision API 提取: 脸型、肤色、五官 → 构造查询
                                              ↓
知识库文档 ← 向量检索 (top-k) ← Embedding(查询)
                                              ↓
LLM (query + retrieved docs + 分析结果) → 个性化妆容推荐
```

### 与评分标准的对应

- **RAG**：体现“检索增强生成”，利用结构化美妆知识提升推荐准确性和可解释性
- **Responsible AI**：知识库可审核、可追溯，减少“随口胡说”；可注明信息来源

## 五、负责任 AI 考量

| 方面 | 措施 |
|------|------|
| **隐私** | 用户照片仅用于分析，不长期存储；明确隐私政策 |
| **偏见** | 知识库覆盖多种肤色、脸型；prompt 中强调包容性 |
| **可解释性** | 推荐时附带理由（如“根据您的鹅蛋脸型，推荐……”） |
| **安全** | 上传限制（格式、大小）；防止滥用；API 限流 |

## 七、实施步骤（MVP 开发顺序）

1. **准备 RAG 知识库**：撰写/整理美妆文档，分块，生成 embeddings 存入 ChromaDB
2. **搭建后端**：FastAPI + 图片上传接口 + Vision 分析 + RAG 检索 + LLM 推荐
3. **搭建前端**：上传界面 + 调用 API + 展示推荐结果
4. **联调与测试**：端到端流程、错误处理、基础限流
5. **（可选）虚拟试妆**：接入图像编辑/试妆 API，作为加分项

## 九、与课堂内容的对应（用于报告/展示）

- **RAG**：美妆知识库 + 向量检索 + 增强 LLM 推荐

- **AI 辅助开发**：用 Cursor/ChatGPT 等辅助前端、后端、RAG 实现
